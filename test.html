Flowchart Audit Washing — Trigger + request_id

Flowchart ini menunjukkan: 1x hit API create washing → insert Washing_h + banyak Washing_d → trigger otomatis simpan log ke AuditTrail. Untuk tampilan, data digroup dengan RequestId dan join ke MstJenisPlastik.

FC Flowchart (Bentuk Diagram)

User / UI Hit API

Kirim payload: header + details (sak).

Backend Service BEGIN TX

Validasi + tutup transaksi check.

Set Context Audit SESSION_CONTEXT

Set actor & request_id supaya trigger tahu “siapa” & bisa grouping.

Insert Header Washing_h

Insert 1 row header (NoWashing, IdJenis, dst).

Trigger Header AuditTrail

Trigger menulis 1 log: INSERT Washing_h (NewData JSON).

Loop Details Washing_d

Per sak: INSERT 1 row detail (NoSak, Berat).

Trigger Detail AuditTrail

Per sak: trigger tulis 1 log. Total = jumlah sak.

Show Data UI View

Ambil audit by RequestId → header + detail. Join MstJenisPlastik untuk tampilkan nama jenis.

Step penting: set actor & request_id
Operasi data utama (insert Washing)
Audit otomatis (trigger → AuditTrail)
Catatan: 1 hit API bisa menghasilkan banyak baris audit untuk Washing_d karena detail disimpan per sak. Untuk UX, tampilkan sebagai “1 transaksi” dengan GROUP BY RequestId.
